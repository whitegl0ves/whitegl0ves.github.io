2、接下来就是具体的 mount 实现

  1. 创建根组件的 vnode

vnode === Virtual DOM，就是离屏渲染的一种设计，本质是一个结构体

createVNode，根节点创建成 vnode ：

params1type是节点类型
例：shapeFlag === STATEFUL_COMPONENT，代表的是一个有状态组件对象
如果传函数，则是函数式组件 FUNCTIONAL_COMPONENT

  1. VNode 渲染成真实的组件

mount 后就是对 vnode 的渲染工作：

初始化执行 patch ，初始化算作全量更新，其他更新则是局部：

patch 参数：

  1. n1 表示老的 vnode 节点（初始化则为空）；
  2. n2 表示新的 vnode 节点；
  3. container 表示需要挂载的 dom 容器；
  4. anchor 挂载的参考元素；
  5. parentComponent 父组件。

根据n2类型判断更新节点类型，进行不同操作

这里需要注意的是 ShapeFlags 是一个二进制左移操作符生成的对象，其中

ShapeFlags.COMPONENT = ShapeFlags.STATEFUL_COMPONENT | ShapeFlags.FUNCTIONAL_COMPONENT，
所以 shapeFlag & ShapeFlags.COMPONENT 这里的值是 true

初始化n1 此时还是个空值，所以进 mountComponent ，函数对组件进行初始挂载过程。

    1. 第二步是初始化组件实例的一些对象，就是传入 setup 上下文：

调用 renderComponentRoot 来生成 subTree，然后再把 subTree 挂载到 container 中。其实 renderComponentRoot 的核心工作就是执行 instance.render 方法：

本质还是调用 createBaseVNode 创建 vnode。所以 subtree 就是调用 render 函数而生产的 vnode 节点。subtree的 shapeFlags === ELEMENT

所以后续调用 processElement 这个函数：

processElement 的实现：

初始化，n1 === null，所以这里执行的是 mountElement 进行元素的初始化挂载。

mountElemet 首先是通过 hostCreateElement 创建了一个 DOM 节点，然后处理一下 props 属性，接着根据 shapeFlag 判断子节点的类型，如果节点是个文本节点，则直接创建文本节点，如果子节点是个数组，比如这种情况：
